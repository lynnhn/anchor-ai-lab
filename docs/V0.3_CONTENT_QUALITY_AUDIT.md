# Anchor AI Lab v0.3 Content Quality Audit

## 1. 总体判断

当前 Astro 内容页已经具备公开访问、路由分享和首页卡片承接能力，但正文层面多数仍属于“占位摘要”状态，不足以作为长期沉淀页面。

主要问题不是视觉或架构，而是内容密度不足：多数页面只有 1-2 段正文，没有清晰的背景、为什么做、当前状态、后续计划和与 Anchor AI Lab 的关系。对于外部访问者来说，可以知道“这个项目/文章大概是什么”，但很难理解“为什么重要、现在做到哪一步、下一步会怎样”。

参考 Downloads 中的 DOCX 资料后判断：ACE、三篇 ACE Writing、AI Tool Reviews、TemuCanvas、AI PPT Workflow 都有可补强素材。其中 ACE 和三篇 Writing 的资料最完整，适合作为 v0.3 第一批内容质量提升的主线。

## 3. 建议 v0.3 第一批修改范围

建议第一批只做内容补强，不改视觉、不改路由、不改首页结构。

1. 补强 `projects/ace`
   - 这是 Anchor AI Lab 的方法论核心。
   - 资料最完整，补完后能反向支撑 Projects、Writing 和 Tool Reviews。

2. 补强 3 篇 ACE Writing
   - `ACE 是什么`
   - `我如何用 Agent 协作完成真实项目`
   - `为什么普通人也需要一套 AI 工程协作方法`
   - 这三篇能形成清晰的公开内容主线：概念定义、实践方式、普通人为什么需要。

3. 补强 `tools/ai-tool-reviews`
   - 现有 docs 已经有比较完整的结构。
   - 适合从兼容 Markdown 迁移为正式 Tools 详情页内容。

4. 补强 `projects/temucanvas` 和 `projects/ai-ppt-workflow`
   - 作为 ACE 的实践样本。
   - 内容应偏公开说明和流程抽象，避免直接搬内部生产手册。

## 4. 不建议现在修改的内容

- 不建议改首页布局或视觉系统。
- 不建议新增 CMS、后台、评论、阅读量或统计。
- 不建议扩展新分类或新路由。
- 不建议把 DOCX 原文整段搬进页面。
- 不建议把 TemuCanvas 写成对外产品官网。
- 不建议把 AI PPT Workflow 写成教程全文，应该先写成项目页和方法摘要。
- 不建议在 v0.3 同时处理 Logo、图片、动效或交互优化。

## 5. 是否建议先补 ACE / AI Tool Reviews / TemuCanvas / AI PPT Workflow

建议优先级如下：

1. **先补 ACE**：必须先做。ACE 是 Anchor AI Lab 的底层方法论，当前详情页过短，和已有资料落差最大。
2. **再补三篇 ACE Writing**：必须跟进。它们能把 ACE 从项目名变成可理解的方法论叙事。
3. **再补 AI Tool Reviews**：应该补。它已有结构化文档，迁移成本低，能提升 Tools 区可信度。
4. **再补 TemuCanvas**：应该补，但要做公开版提炼，避免过多内部生产细节。
5. **最后补 AI PPT Workflow**：应该补。资料很充足，但原资料更像系统教程，v0.3 先提炼为项目说明，不宜全文教程化。
