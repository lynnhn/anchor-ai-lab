# Anchor AI Lab v0.4 Public Polish Audit

## 1. 总体判断

v0.3 内容质量已经明显提升，当前站点适合作为公开个人 AI 实践站继续迭代。首页、Projects、Writing、Tools 详情页和旧 Markdown 兼容链接都能公开访问。

但从“公开上线基础要素”看，当前仍偏向内容站骨架阶段。最需要补的是 SEO / 社交分享 / 404 / README 公开部署信息，而不是继续大改内容或视觉。

当前最明显的缺口是：页面只有基础 title、description、viewport 和 favicon 占位；没有 Open Graph、Twitter Card、canonical、真实 robots.txt、sitemap.xml 和 404 页面。线上访问 `robots.txt`、`sitemap.xml` 和不存在路径时都会返回首页 200，说明这些基础公开访问规则还没有建立。

## 3. 缺失项

### Meta / Social

- `html` 当前是 `lang="zh"`，建议改为 `lang="zh-CN"`。
- `description` 是全站固定描述，详情页没有按内容生成独立 description。
- 缺少 `og:title`。
- 缺少 `og:description`。
- 缺少 `og:type`。
- 缺少 `og:url`。
- 缺少 `og:image`。
- 缺少 Twitter Card：
  - `twitter:card`
  - `twitter:title`
  - `twitter:description`
  - `twitter:image`
- 缺少 canonical link。
- `Layout.astro` 当前 favicon 仍是 `href="data:,"`，没有实际引用 `public/favicon.ico` 或 `public/favicon.svg`。

### SEO / Crawlers

- 未发现真实 `robots.txt`。
- 未发现真实 `sitemap.xml`。
- 线上访问 `/robots.txt` 返回首页 HTML 200，不是真实 robots 文件。
- 线上访问 `/sitemap.xml` 返回首页 HTML 200，不是真实 sitemap。
- 不存在路径返回首页 200，说明当前缺少真实 404 行为或 Pages fallback 规则需要进一步确认。

### 404

- 未发现 `anchor-ai-lab-astro/src/pages/404.astro`。
- 建议新增最小 404 页面，复用 Layout / Header / Footer，不做复杂设计。

### README / Deployment Info

- README 还没有写线上地址：`https://anchor-ai-lab.pages.dev/`。
- README 还没有写 Cloudflare Pages 配置：
  - Framework preset: Astro
  - Root directory: `anchor-ai-lab-astro`
  - Build command: `npm run build`
  - Build output directory: `dist`
  - Production branch: `main`

## 4. 建议 v0.4 第一批最小修改

1. 补强 `Layout.astro` 的公开站点 meta
   - 支持 `title`、`description`、`canonical`、`og:*`、`twitter:*` props。
   - 将 `lang="zh"` 改为 `lang="zh-CN"`。
   - 将 favicon 指向实际文件，如 `/favicon.svg` 和 `/favicon.ico`。

2. 让详情页传入页面级 description 和 canonical
   - Projects / Writing / Tools 详情页使用 `entry.data.summary` 作为 description。
   - canonical 使用线上域名和当前路径。
   - 首页使用站点默认 description。

3. 新增基础 SEO 文件
   - `public/robots.txt`
   - `public/sitemap.xml`
   - 当前页面数量很少，可以先手写 sitemap，后续再考虑自动生成。

4. 新增最小 404 页面
   - `src/pages/404.astro`
   - 保持现有视觉气质，只说明页面不存在并提供返回首页链接。

5. 补 README 公开部署信息
   - 添加线上地址。
   - 添加 Cloudflare Pages UI 配置摘要。
